# Dracul [drei] 開発計画書

資料「Dracul_[drei] 総合設定まとめ.txt」に基づき、本プロジェクトの要件定義および1年間の開発スケジュールを策定しました。

## 1. プロジェクトコンセプト
*   **タイトル:** Dracul [drei] (仮)
*   **ジャンル:** ポストアポカリプス・ダークファンタジー・アクション
*   **プラットフォーム:** PC (Windows)
*   **エンジン:** Unity
*   **コアテーマ:** 滅びた世界における「人口調整装置」としての生存。「食事としての戦闘」。

## 2. 要件定義

### 2.1. プレイヤーキャラクター (ドラキュラ)
*   **正体:** 人造の存在（「ノスフェラトゥ」）。感情を持たず、知識のみを持つ。
*   **基本ステータス:**
    *   **ブラッドゲージ (Blood Gauge):** エネルギー兼空腹度。
        *   時間経過や行動（ダッシュ等）で減少。
        *   死体からの「吸血（給血）」または輸血パックで回復。
        *   **枯渇状態:** 移動速度低下、攻撃不可。
    *   **耐久度 (Durability / Health):**
        *   被ダメージで減少。
        *   ダメージは出血（ブラッドゲージ減少の加速）を伴う。
        *   ブラッドゲージが十分にあれば時間経過で自然治癒。
*   **弱点:** 日光。
    *   「焼損」（耐久度減少）と「弱体化」（ステータス低下）を引き起こす。
    *   重度の弱体化で攻撃不可になる。
*   **成長・進行:**
    *   **初期:** 基本移動、近接攻撃（爪/素手）。
    *   **解放:** ダッシュ、銃（マテバ風）、サーベル、乗り物（車/戦車）。
    *   **知識:** 人類の記録を読むことで知識（ステータス/スキル）を得る。

### 2.2. 世界観・環境
*   **設定:** 核戦争後の地球。人類絶滅済み。
*   **雰囲気:** 廃墟、文明の痕跡、汚染された自然。
*   **昼夜サイクル:** ゲームプレイに直結（夜は安全、昼は危険）。
*   **存在:**
    *   **メガ進化生物:** プレイヤーにとっての「食料」。
    *   **元人間（変異体）:** 敵対的。意思疎通不可。
    *   **機械/AI:** 暴走した遺物。

### 2.3. ゲームループ
1.  **探索:** 廃墟を彷徨う（三人称視点）。
2.  **生存:** ブラッドゲージと昼夜サイクルを管理する。
3.  **狩り:** 敵（食料）を倒し、血を補給する。
4.  **成長:** 記録を見つけ、ドラキュラを強化する。

## 3. 技術戦略とアセット
*   **目標:** Unityの機能を活かし、低負荷・低工数で効率的に開発する。

### 3.1. 推奨Unity機能 (標準/無料パッケージ)
*   **ProBuilder:** 外部ツール不要で、素早くグレーボクシング（仮レベル作成）を行う。
*   **Terrain Tools:** 荒野の環境作成に使用。
*   **NavMesh:** 敵AIの経路探索に使用。
*   **Input System (New):** コントローラー/キーボード設定の簡易化。

### 3.2. 推奨アセット (無料/オープンソース中心)
コストを抑え、開発速度を上げるための提案：

*   **キャラクターコントローラー:**
    *   *提案:* **Starter Assets - Third Person Character Controller** (Unity Technologies - 無料)
    *   *理由:* 信頼性が高く、移動・ジャンプ等の基本動作が実装済み。
*   **3Dモデル (キャラクター/敵):**
    *   *提案:* **Mixamo** (Adobe - 無料)
    *   *理由:* アニメーションとリグのライブラリが豊富。「元人間」のベースに最適。
    *   *提案:* **Robot Kyle** (Unity - 無料) 等の基本的なロボットモデル（メカ敵用）。
*   **環境:**
    *   *提案:* **Flooded Grounds** (Asset Store無料枠) または **Post Apocalyptic World** (無料版/サンプル)。
    *   *提案:* **Free Nature/Rock Packs** (Book of the Dead等は高品質だが重いため、Low Poly系の自然パックも検討)。
*   **UI:**
    *   *提案:* **Kenney Assets (UI Pack)** (無料)
    *   *理由:* シンプルで汎用性が高く、ダークテーマに合わせやすい。

## 4. 年間開発スケジュール

### Phase 1: 基盤構築 (1ヶ月目 - 3ヶ月目)
**目標:** 基本移動とステータスを備えたプレイアブルなプロトタイプ。
*   **1ヶ月目:** プロジェクトセットアップ。Third Person Controller導入。ProBuilderでテストマップ作成。
*   **2ヶ月目:** **ブラッドゲージ**と**待機時減少**ロジックの実装。**昼夜サイクル**システムの実装。
*   **3ヶ月目:** **日光ダメージ**処理（太陽からプレイヤーへのレイキャスト判定）。ヘルス/ブラッドUIの作成。

### Phase 2: コアメカニクス (4ヶ月目 - 6ヶ月目)
**目標:** 戦闘とゲームループの実装。
*   **4ヶ月目:** 戦闘システムの基礎。近接攻撃判定。「吸血」インタラクション（アニメーション＋回復処理）。
*   **5ヶ月目:** 敵AI v1（徘徊 > 追跡 > 攻撃）。NavMesh利用。「元人間」敵の実装。
*   **6ヶ月目:** ダメージシステム（耐久度 vs ブラッド減少）。出血状態異常。死亡/リスポーン処理。

### Phase 3: 世界構築と成長 (7ヶ月目 - 9ヶ月目)
**目標:** コンテンツ拡充と成長要素。
*   **7ヶ月目:** 本番マップ作成。Terrain造成。廃墟/建物の配置。
*   **8ヶ月目:** 成長システム。「記録を読む」UIとステータス上昇処理。ダッシュ機能の解放。
*   **9ヶ月目:** 武器バリエーション（銃/サーベルの実装）。上位敵（メカ/ロボット）の実装。

### Phase 4: ポリッシュと仕上げ (10ヶ月目 - 12ヶ月目)
**目標:** 最適化と「ゲームの手触り」向上。
*   **10ヶ月目:** 乗り物システム（車）の試作。*※スケジュールが厳しい場合は簡易化またはカット。*
*   **11ヶ月目:** UI/UXの洗練。効果音（環境音、環境風、戦闘音）。メインメニュー/ポーズ画面。
*   **12ヶ月目:** バグ修正。パフォーマンス・プロファイリング（メモリ/FPS）。最終バランス調整。

## 5. 開発ガイドライン
*   **シンプルに保つ:** あらゆる要素に複雑なシステムを組まない。ステータスは単純なfloat値で管理する。
*   **スクリプトのモジュール化:** `PlayerHealth`, `PlayerMovement`, `DayNightManager` 等、機能ごとに分離する。
*   **プロトタイピング:** 「吸血」の感触は良いか？ グラフィックよりも先にコアループの「手触り」を優先する。
